# 2.6 O que é importante levar deste capítulo

Neste capítulo, não é necessário memorizar todos os termos ou decorar cada detalhe técnico. O que realmente importa é entender **para que serve um sistema de controle de versão**, **por que ele foi criado** e **qual a diferença principal entre os dois tipos de modelos**: centralizado e distribuído.

O modelo centralizado apresenta algumas limitações, principalmente quando várias pessoas trabalham ao mesmo tempo ou quando não há acesso contínuo ao servidor. O modelo distribuído surgiu para superar essas limitações, permitindo que cada pessoa tenha uma cópia completa do projeto, trabalhe de forma independente, registre alterações e sincronize tudo com o repositório remoto quando quiser.

O que você deve levar daqui é essa visão geral: o modelo distribuído existe para facilitar a colaboração, aumentar a flexibilidade e reduzir os problemas do modelo centralizado. Ao decorrer dos próximos capítulos, vamos aprender como isso funciona na prática usando o Git, vendo passo a passo como criar, modificar e sincronizar versões de um projeto.
