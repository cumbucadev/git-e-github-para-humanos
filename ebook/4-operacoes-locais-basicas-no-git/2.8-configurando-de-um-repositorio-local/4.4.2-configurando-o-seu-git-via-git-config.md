# 4.4.2 Configurando o seu Git via \`git config\`

Agora que você já entendeu como funcionam os escopos de configuração do _Git_, é hora de ver como gerenciar essas configurações na prática usando a linha de comando.

Embora as configurações do _Git_ sejam armazenadas em arquivos no sistema, **não é recomendável editá-los manualmente**. Esses arquivos existem para armazenar as configurações que o Git utiliza no dia a dia.

Nesta seção, vamos observar esses arquivos em alguns momentos para entender melhor o comportamento do Git e a forma como ele organiza suas configurações internamente. No entanto, a criação, alteração e remoção de configurações devem ser feitas sempre por meio da linha de comando.

É justamente para isso que existe o comando <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark>. Ele permite que você gerencie configurações pela _CLI_, garantindo que os valores sejam gravados no arquivo correto, no escopo adequado, e evitando erros de formatação ou inconsistências.

## git config

O comando <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> é usado para configurar o Git no seu computador. Neste contexto, configurar significa customizar ou personalizar o Git de acordo com suas preferências e necessidades.

Com esse comando, você consegue, entre outras coisas, listar configurações existentes, consultar valores específicos, alterar informações e remover configurações já definidas. A seguir, veremos esses principais casos de uso.

### Listar configurações

Esse uso é indicado quando você quer visualizar quais configurações existem e quais valores estão ativos no momento.

#### Estrutura

> <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--list</mark> <mark style="color:blue;">\[opções]</mark>

Em que:

* <mark style="color:blue;">**--list**</mark>: indica que o Git deve listar as configurações disponíveis.
* <mark style="color:blue;">**\[opções]**</mark> **(opcional)**: define modificadores adicionais do comando, como filtrar por escopo ou exibir a origem das configurações.

#### Exemplos de uso

* Lista todas as configurações visíveis para o Git, considerando todos os escopos
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--list</mark>
* Listar as configurações e mostra de qual arquivo cada uma está sendo lida
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--list --show-origin</mark>
* Lista apenas as configurações definidas no escopo system
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--list --system</mark>
* Listar apenas as configurações definidas no escopo global
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--list --global</mark>
* Lista apenas as configurações definidas no repositório atual
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--list --local</mark>

Quando uma configuração existe em mais de um escopo, o Git considera apenas o valor que está efetivamente ativo.

### Saber o valor de uma configuração

Esse uso é indicado quando você quer consultar o valor atual de uma configuração específica.

#### Estrutura

> <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">\[escopo]</mark> <mark style="color:green;">chave</mark>

Em que:

* <mark style="color:blue;">**\[escopo]**</mark> **(opcional)**: define o escopo da consulta, como `--global`, `--local` ou `--system`.
* <mark style="color:green;">**chave**</mark>: se refere ao nome da configuração que você deseja consultar.\
  Exemplos: `user.name`, `user.email`.

#### Exemplos de uso

* Mostra o valor atualmente ativo da configuração `user.name`
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:green;">user.name</mark>
* Mostra o valor de `user.name` definido no escopo system
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--system</mark> <mark style="color:green;">user.name</mark>
* Mostra o valor de `user.name` definido no escopo global
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--global</mark> <mark style="color:green;">user.name</mark>
* Mostra o valor de `user.name` definido no repositório atual
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--local</mark> <mark style="color:green;">user.name</mark>

### Alterar ou definir uma configuração

Esse uso é indicado quando você quer criar uma nova configuração ou alterar o valor de uma configuração existente.

#### Estrutura

> <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">\[escopo]</mark> <mark style="color:green;">chave valor</mark>

Em que:

* <mark style="color:blue;">**\[escopo]**</mark> **(opcional)**: define o escopo em que a configuração será aplicada.
* <mark style="color:green;">**chave**</mark>: se refere ao nome da configuração que você deseja definir ou modificar.\
  Exemplos: `user.name`, `user.email`.
* <mark style="color:green;">**valor**</mark>: se refere ao valor que você deseja atribuir à chave.\
  Exemplos: `Aprendiz Cumbuca Dev`, `aprendiz.cumbucadev@gmail.com`.\
  Quando o valor contém espaços, ele deve ser colocado entre aspas. Isso indica ao _Git_ que todo o texto faz parte de um único valor, e não de vários argumentos separados.

#### Exemplos de uso

* Define o valor de `user.name` no escopo padrão
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:green;">user.name</mark> <mark style="color:green;">"Aprendiz Cumbuca Dev"</mark>
* Define o valor de `user.name` no escopo system
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--system</mark> <mark style="color:green;">user.name</mark> <mark style="color:green;">"Aprendiz Cumbuca Dev"</mark>
* Define o valor de `user.name` no escopo global
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--global</mark> <mark style="color:green;">user.name</mark> <mark style="color:green;">"Aprendiz Cumbuca Dev"</mark>
* Define o valor de `user.name` apenas para o repositório atual
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--local</mark> <mark style="color:green;">user.name</mark> <mark style="color:green;">"Aprendiz Cumbuca Dev"</mark>

Quando uma configuração existe em mais de um escopo, o Git utiliza sempre o valor definido no escopo mais específico.

### Apagar uma configuração

Esse uso é indicado quando você quer remover uma configuração de um determinado escopo.

#### Estrutura

> <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">\[escopo] --unset</mark> <mark style="color:green;">chave</mark>

Em que:

* <mark style="color:blue;">**\[escopo]**</mark> **(opcional)**: define o escopo da remoção.
* <mark style="color:blue;">**--unset**</mark>: indica que a configuração deve ser removida.
* <mark style="color:green;">**chave**</mark>: se refere ao nome da configuração que será apagada.\
  Exemplos: `user.name`, `user.email`.

#### Exemplos de uso

* Remove a configuração `user.name` do escopo padrão
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--unset</mark> <mark style="color:green;">user.name</mark>
* Remove a configuração `user.name` apenas do repositório atual
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--local --unset</mark> <mark style="color:green;">user.name</mark>
* Remove a configuração `user.name` do escopo global
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--global --unset</mark> <mark style="color:green;">user.name</mark>
* Remove a configuração `user.name` do escopo system.
  * <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark> <mark style="color:blue;">--system --unset</mark> <mark style="color:green;">user.name</mark>

Após remover uma configuração, o Git passa a considerar o valor definido em um escopo mais amplo, caso ele exista.

***

Com isso, você já conhece os principais usos do comando <mark style="color:purple;">git</mark> <mark style="color:orange;">config</mark>, como o Git organiza suas configurações em diferentes escopos e como esses valores são armazenados e lidos internamente.

A seguir, vamos ver um exemplo completo de como essas configurações aparecem no dia a dia e por que elas são necessárias logo nos primeiros passos ao trabalhar com _Git_.
