# 4.8 Glossário: Capítulo 4

Estes são os termos que consideramos importantes na compreensão total do Capítulo 4. Ao final do curso, você encontrará um glossário completo com todos os termos que consideramos essenciais!

| Termo                        | Definição (no contexto do Capítulo 4)                                                       |
| ---------------------------- | ------------------------------------------------------------------------------------------- |
| **Repositório Git**          | “Pasta especial” que guarda os arquivos do projeto e o histórico de mudanças.               |
| **Repositório local**        | Repositório que fica no seu computador. É onde você trabalha no dia a dia.                  |
| **Repositório remoto**       | Repositório hospedado em um servidor. É usado para compartilhar e sincronizar.              |
| **`git init`**               | Comando que cria um novo repositório Git no diretório atual (ou em um diretório informado). |
| **Diretório `.git`**         | Pasta oculta criada pelo Git. Guarda metadados, histórico e configurações do repositório.   |
| **Configurações do Git**     | Preferências que mudam o comportamento do Git. Ex.: identidade, formato de saída e atalhos. |
| **`git config`**             | Comando para listar, consultar, definir e remover configurações do Git.                     |
| **Escopo de configuração**   | “Alcance” de uma configuração. Define para quem ela vale e onde o Git deve buscá-la.        |
| **Escopo&#x20;**_**system**_ | Configurações para todo o computador, valendo para todas as pessoas usuárias.               |
| **Escopo&#x20;**_**global**_ | Configurações da pessoa usuária, valendo para todos os seus repositórios.                   |
| **Escopo&#x20;**_**local**_  | Configurações de um repositório específico. Ficam em `.git/config`.                         |
| **Prioridade de escopos**    | Quando há conflito, o Git usa o valor mais específico: _local_ > _global_ > _system_.       |
| **`user.name`**              | Nome que o Git grava como autoria dos commits.                                              |
| **`user.email`**             | Email que o Git grava como autoria dos commits.                                             |

{% hint style="danger" %}
Atenção: os termos selecionados são explicados de acordo com o contexto específico de _**cada capítulo**_**.**
{% endhint %}
