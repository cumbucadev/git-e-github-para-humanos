# 1.3 Familiarizando-se com o terminal

Agora que já falamos sobre dar ordens ao computador por meio de linhas de comando, é hora de conhecer a ferramenta onde essa interação acontece: o **terminal**.

O terminal é, essencialmente, uma janela que permite conversar com o sistema operacional usando texto. Dependendo do sistema e do contexto, ele também pode aparecer com outros nomes, como _prompt de comando_, _console_, _linha de comando_ ou _interface de linha de comando_. Neste livro, vamos usar o termo **terminal**, que é o mais comum e o que você provavelmente vai encontrar no seu computador. Ele recebe o que você digita, envia isso para o sistema operacional e mostra de volta o resultado. É uma ferramenta simples na aparência, mas muito poderosa no que permite fazer.

Talvez você reconheça o terminal como aquela “tela preta” que aparece em filmes e séries quando alguém está programando ou quando um hacker está fazendo algo misterioso. Em muitos casos, ele realmente aparece assim: uma janela escura, com texto claro, sem botões, sem menus e sem ícones. Esse visual costuma causar estranhamento no início, mas ele existe justamente para reduzir distrações e deixar o foco na interação direta.

Quando você abre um terminal, algumas informações básicas costumam aparecer. Mesmo sem entender tudo ainda, é importante saber que elas ajudam a situar você naquele momento. Uma delas é o **local atual**, também chamado de caminho ou _path_. Ele indica em que pasta do computador você está trabalhando. No terminal, sempre existe a ideia de “onde você está” dentro do sistema de arquivos.

Outro conceito importante é o de **entrada e saída**. Tudo o que você digita no terminal é uma entrada: são as ordens que você está dando. Tudo o que o terminal mostra depois disso é a saída: mensagens, resultados, avisos ou erros. Essa troca constante — você envia algo, o computador responde — é a base da interação via terminal.

Esses conceitos são simples, mas fundamentais. Entender que você está sempre em um lugar específico do computador, que envia entradas e recebe saídas, ajuda muito a não se sentir perdida ao usar o terminal.

A forma de abrir o terminal varia de acordo com o sistema operacional. No macOS e em distribuições Linux, ele costuma estar disponível como um aplicativo chamado “Terminal”. No Windows, existem algumas opções, como o Windows Terminal ou o Prompt de Comando. O caminho até ele pode mudar, mas a ideia é sempre a mesma: abrir uma janela onde você possa digitar comandos.

Também existem diferenças entre os sistemas operacionais quando falamos dos comandos em si. De forma geral, macOS e Linux compartilham muitos comandos em comum, enquanto o Windows usa alguns nomes e comportamentos diferentes. Essas diferenças existem, mas não precisam ser motivo de preocupação agora. Ao longo do livro, vamos deixar claro quando algo muda de um sistema para outro.

A partir daqui, vamos começar pelo básico: aprender como abrir o terminal em cada sistema operacional. Esse será o primeiro passo prático para colocar tudo isso em ação.
