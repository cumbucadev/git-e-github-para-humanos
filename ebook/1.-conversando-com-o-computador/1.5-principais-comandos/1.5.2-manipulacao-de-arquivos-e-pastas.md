# 1.4.2 Manipulação de arquivos e pastas

Agora que você já sabe **onde está** no sistema de arquivos e como se mover entre pastas, é hora de aprender a **manipular arquivos e pastas**: criar, copiar, mover, renomear e excluir.

Essas são ações básicas do dia a dia. Na interface gráfica, você faz isso clicando, arrastando ou usando menus. No terminal, fazemos exatamente as mesmas coisas — mas usando **comandos de texto**, com mais controle e precisão.

Como antes, **não tente decorar tudo agora**. O objetivo é entender o que cada comando faz e reconhecer seus nomes. Com o uso, eles se tornam naturais.

## Principais comandos

### 1. Criar arquivos

**Comando no Linux/macOS:**

```bash
touch nome-do-arquivo.txt
```

**Comando no Windows (CMD):**

```cmd
type nul > nome-do-arquivo.txt
```

**Comando no Windows (PowerShell):**

```powershell
New-Item nome-do-arquivo.txt
```

**O que faz:**\
Cria um arquivo vazio no diretório atual.

**Saída:**\
Nenhuma.

**Exemplo de comando:**

```bash
touch notas.txt
```

### 2. Criar pastas

**Comando no Linux/macOS/Windows (CMD/PowerShell):**

```bash
mkdir nome-da-pasta
```

**O que faz:**\
Cria uma nova pasta no diretório atual.

**Saída:**\
Nenhuma.

**Exemplo de comando:**

```bash
mkdir projetos
```

### 3. Copiar arquivos

**Comando no Linux/macOS:**

```bash
cp arquivo-origem arquivo-destino
```

**Comando no Windows (CMD/PowerShell):**

```cmd
copy arquivo-origem arquivo-destino
```

**O que faz:**\
Cria uma cópia de um arquivo, mantendo o original intacto.

**Saída:**\
Nenhuma.

**Exemplo de comando:**

```bash
cp notas.txt backup-notas.txt
```

### 4. Copiar pastas

**Comando no Linux/macOS:**

```bash
cp -r pasta-origem pasta-destino
```

**Comando no Windows (CMD/PowerShell):**

```cmd
xcopy pasta-origem pasta-destino /E
```

**O que faz:**\
Copia uma pasta e todo o seu conteúdo.

**Saída:**\
Nenhuma.

**Exemplo de comando:**

```bash
cp -r projetos projetos-backup
```

### 5. Mover ou renomear arquivos e pastas

**Comando no Linux/macOS:**

```bash
mv origem destino
```

**Comando no Windows (CMD/PowerShell):**

```cmd
move origem destino
```

**O que faz:**\
Move arquivos ou pastas para outro local ou altera seus nomes.

**Saída:**\
Nenhuma.

**Exemplo de mover arquivo:**

```bash
mv notas.txt projetos/
```

**Exemplo de renomear arquivo:**

```bash
mv notas.txt ideias.txt
```

### 6. Excluir arquivos

**Comando no Linux/macOS:**

```bash
rm nome-do-arquivo
```

**Comando no Windows (CMD/PowerShell):**

```cmd
del nome-do-arquivo
```

**O que faz:**\
Remove definitivamente um arquivo.

**Saída:**\
Nenhuma.

**Exemplo de comando:**

```bash
rm ideias.txt
```

⚠️ O arquivo é removido diretamente, sem ir para a lixeira.

### 7. Excluir pastas

**Comando no Linux/macOS:**

```bash
rm -r nome-da-pasta
```

**Comando no Windows (CMD/PowerShell):**

```cmd
rmdir /S nome-da-pasta
```

**O que faz:**\
Remove uma pasta e todo o seu conteúdo.

**Saída:**\
Nenhuma.

**Exemplo de comando:**

```bash
rm -r projetos-antigos
```

### Atenção ao usar comandos de remoção

Os comandos de remoção (`rm`, `del`, `rmdir`) **não pedem confirmação por padrão** e **não usam a lixeira**. A exclusão é imediata.

Antes de remover qualquer coisa, é uma boa prática:

* conferir onde você está (`pwd`);
* listar o conteúdo (`ls` ou `dir`);
* ter certeza do que está prestes a apagar.

Mesmo pessoas experientes fazem isso o tempo todo.
