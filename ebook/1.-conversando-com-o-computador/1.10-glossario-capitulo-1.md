# 1.10 Glossário: Capítulo 1

Estes são os termos que consideramos importantes na compreensão total do Capítulo 1. Ao final do curso, você encontrará um glossário completo com todos os termos que consideramos essenciais!

| Termo                                 | Definição (no contexto do Capítulo 1)                                                         |
| ------------------------------------- | --------------------------------------------------------------------------------------------- |
| **Sistema Operacional (SO)**          | Software “base” do computador. Ele coordena programas, arquivos e dispositivos.               |
| **Sistema de arquivos**               | Forma como o SO organiza e encontra arquivos e pastas. Funciona como uma árvore.              |
| **Arquivo**                           | Unidade de informação armazenada. Ex.: texto, imagem, código.                                 |
| **Pasta / Diretório**                 | Contêiner para organizar arquivos e outras pastas.                                            |
| **Raiz (root)**                       | Pasta inicial da árvore de arquivos. No Linux/macOS é `/`.                                    |
| **Diretório atual**                   | Pasta em que você está “posicionado” no terminal. Quase todo comando depende disso.           |
| **Diretório acima (diretório pai)**   | Pasta imediatamente acima na hierarquia. Você acessa com `..`.                                |
| **Caminho (path)**                    | “Endereçoço de um arquivo/pasta no sistema de arquivos.                                       |
| **Caminho absoluto**                  | Caminho completo desde a raiz. Ex.: `/home/usuario/notas.txt` ou `C:\Users\...`.              |
| **Caminho relativo**                  | Caminho a partir do diretório atual. Ex.: `projetos/notas.txt` ou `..\app.txt`.               |
| **Linha de comando**                  | Forma de dar ordens por texto. Você escreve instruções, o sistema executa.                    |
| **Terminal**                          | Aplicativo onde você digita comandos e vê o resultado. Também chamado de prompt/console.      |
| **Entrada**                           | O que você digita no terminal antes de pressionar `Enter`.                                    |
| **Saída**                             | Resposta exibida pelo terminal após executar um comando. Pode não existir.                    |
| **Mensagem de erro**                  | Saída que indica que o comando não foi executado como esperado.                               |
| **`pwd`**                             | Mostra o caminho do diretório atual.                                                          |
| **`ls` / `dir`**                      | Lista arquivos e pastas no diretório atual (Linux/macOS: `ls`; Windows: `dir`).               |
| **`cd`**                              | Muda de diretório (entra/sai de pastas). Ex.: `cd projetos`, `cd ..`, `cd ~`.                 |
| **`touch` / `type nul` / `New-Item`** | Cria arquivo vazio (Linux/macOS: `touch`; Windows CMD: `type nul >`; PowerShell: `New-Item`). |
| **`mkdir`**                           | Cria uma pasta.                                                                               |
| **`cp` / `copy`**                     | Copia arquivos (Linux/macOS: `cp`; Windows: `copy`).                                          |
| **`cp -r` / `xcopy /E`**              | Copia pastas com conteúdo (Linux/macOS: `cp -r`; Windows: `xcopy ... /E`).                    |
| **`mv` / `move`**                     | Move ou renomeia arquivos/pastas.                                                             |
| **`rm` / `del` / `rmdir`**            | Remove arquivos/pastas. Normalmente não usa lixeira.                                          |

{% hint style="warning" %}
Atenção: os termos selecionados são explicados de acordo com o contexto específico de _**cada capítulo**_**.**
{% endhint %}
