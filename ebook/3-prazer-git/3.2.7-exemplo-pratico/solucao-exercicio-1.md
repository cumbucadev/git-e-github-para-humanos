# Solução Exercício 1

Antes de tudo, certifique-se de que:

* o Git está instalado (como ensinado anteriormente em [2.4-instalando-o-git](../2.4-instalando-o-git/ "mention"));
* o terminal está aberto (como também já aprendemos em [1.4-familiarizando-se-com-o-terminal](../../1.-conversando-com-o-computador/1.4-familiarizando-se-com-o-terminal/ "mention")).



1. Execute no terminal:

```bash
git
```

<figure><img src="../../.gitbook/assets/0274.png" alt=""><figcaption></figcaption></figure>

Isso exibirá uma lista de comandos disponíveis, organizados por categoria, como por exemplo:

* `clone`, `init`       &#x20;
* `add`, `mv`, `restore`,`rm`&#x20;
* entre outros.

2. Identifique **3 comandos que você nunca viu antes**. Por exemplo:

* `bisect`
* `stash`
* `cherry-pick`

3. Identifique **2 comandos que parecem relacionados** (mesma categoria ou objetivo):

* `add` e `mv` (ambos relacionados a registrar mudanças);
* `branch` e `merge` (ambos relacionados a gerenciar ramificações).

Não se preocupe em entender tudo agora. O objetivo é apenas **explorar** e perceber a quantidade de coisas que o Git oferece, reconhecendo o **ponto de entrada da ferramenta**.
